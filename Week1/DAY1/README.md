# Day 1 - Customer Data Cleaning with PySpark

## Objective

Build a reliable cleaning pipeline for a raw customer CSV by handling null-like strings, inconsistent formats, invalid dates, and duplicates.

## Files in This Folder

- `C1.csv` - raw customer source file with quality issues
- `customer_cleaning_pyspark.py` - full PySpark cleaning script
- `DAY1.py` - empty placeholder file
- `__pycache__/customer_cleaning_pyspark.cpython-312.pyc` - auto-generated bytecode

## Raw Data Issues Observed in `C1.csv`

The source data intentionally contains practical problems:
- mixed country representations (`india`, `India`, `New York`, `USA`)
- placeholder text values such as `blank` and `Blank`
- mixed date formats (`dd-MM-yyyy` and `MM-dd-yyyy`)
- duplicate business records (for example repeated Alice row)
- missing or invalid critical fields (for example `JoinDate = blank`)

## Cleaning Pipeline Implemented (`customer_cleaning_pyspark.py`)

### 1. Input/Output setup
- default input: `C1.csv`
- default output: `cleaned_output/`
- accepts optional CLI paths:
  - `python customer_cleaning_pyspark.py <input_path> <output_path>`

### 2. Standardize null-like strings
- trims values
- converts `""`, `blank`, `null`, `none`, `na`, `n/a` to true nulls

### 3. Data type normalization
- `CustomerID` -> `int`
- `Sales` -> `double`

### 4. Categorical normalization
- `Name` -> proper case (`initcap`)
- `Country` mapping:
  - `india` -> `India`
  - `usa` -> `USA`
  - `uk` -> `UK`
  - `new york` -> `USA`
- `Category` validated to `A/B/C`; else `Unknown`

### 5. Date harmonization
- parses `JoinDate` with fallback formats:
  - `dd-MM-yyyy`
  - `MM-dd-yyyy`

### 6. Missing value strategy
- fills nullable non-key fields:
  - `Name`, `Country`, `Category` -> `Unknown`
  - `Sales` -> `0.0`

### 7. Integrity filtering and deduplication
- drops rows missing mandatory columns: `CustomerID`, `JoinDate`
- removes duplicate business records based on:
  - `Name`, `Country`, `JoinDate`, `Sales`, `Category`

### 8. Output
- writes cleaned dataset to CSV (overwrite mode) with header

## Key Concepts Demonstrated

- defensive ingestion patterns
- null standardization before transformation
- schema/type enforcement
- categorical normalization and controlled fallback values
- date parsing with multi-format support
- key-based row rejection and deduplication

## Outcome

Day 1 establishes the foundation for downstream analytics by converting raw customer data into a clean, typed, and analysis-ready dataset.
